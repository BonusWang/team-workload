# 团队工作量管理系统（TeamWorkload）初始化搭建 Spec

## Why
当前项目为空目录，需要从零搭建一个完整的团队工作量管理系统，统一管理开发团队人力与工作量，自动统计工作量分布，识别人力饱和与空缺，支撑团队资源合理调配。

## What Changes
- 搭建 Spring Boot 2.x 后端单体应用（JDK 8 + MyBatis-Plus + MySQL 5.7）
- 搭建 Vue3 前端应用（Element Plus + ECharts + Pinia + Vite）
- 实现用户模块（登录认证、用户 CRUD、角色权限、JWT）
- 实现工作日模块（对接互联网工作日接口、手动调整）
- 实现 Excel 导入模块（全开系统模板 + 其他投入清单模板，统一6字段解析）
- 实现工作量模块（日报数据管理、周报汇总、月度统计）
- 实现请假模块（请假申请、直属上级审批、余额查询）
- 实现看板模块（个人看板、月度日历、团队视图）
- 实现导出模块（周报/月报 Excel 导出）
- 实现团队人力空缺分析
- 前端 Dark Mode 科技感 UI（参考 trae.cn 风格）
- Docker 部署配置

## Impact
- Affected code: 全新项目，无已有代码影响
- 数据库: MySQL 5.7，需创建完整数据库表结构
- 前端: Vue3 SPA 应用
- 后端: Spring Boot 单体服务

## ADDED Requirements

### Requirement: 项目基础框架搭建
系统 SHALL 提供 Spring Boot 2.x 后端项目骨架与 Vue3 前端项目骨架，前后端分离开发，最终打包为单体 Docker 镜像部署。

#### Scenario: 后端项目初始化
- **WHEN** 开发者启动后端项目
- **THEN** Spring Boot 应用正常启动，连接 MySQL 数据库，Swagger/Knife4j 接口文档可访问

#### Scenario: 前端项目初始化
- **WHEN** 开发者启动前端项目
- **THEN** Vite 开发服务器正常启动，Vue3 应用可访问，Element Plus 组件库可用

---

### Requirement: 用户模块
系统 SHALL 提供基于角色的用户管理功能，支持 JWT Token 认证，角色分为普通成员、Leader、管理员。

#### Scenario: 用户登录
- **WHEN** 用户提交用户名和密码
- **THEN** 系统验证凭据，返回 JWT Token，前端存储 Token 并跳转至首页

#### Scenario: 用户管理（管理员）
- **WHEN** 管理员访问用户管理页面
- **THEN** 可查看用户列表、新增/编辑/禁用用户，设置角色、级别、结算单价、归属上级

#### Scenario: 权限控制
- **WHEN** 普通成员访问管理员接口
- **THEN** 系统返回 403 Forbidden

---

### Requirement: 工作日模块
系统 SHALL 对接互联网工作日接口自动获取国家法定工作日数据，并支持管理员手动调整。

#### Scenario: 同步年度工作日
- **WHEN** 管理员触发年度工作日同步
- **THEN** 系统调用外部 API 获取指定年度工作日数据，存储至 WorkDay 表

#### Scenario: 手动调整工作日
- **WHEN** 管理员修改某日的工作日配置（如调休补班）
- **THEN** 系统更新该日记录，影响请假天数计算和日历标记

---

### Requirement: Excel 导入模块
系统 SHALL 支持两种 Excel 模板导入日报数据，统一提取6个字段（所属项目/产品、任务号、任务名称、工作日期、实际工时、提交人）。

#### Scenario: 全开系统模板导入
- **WHEN** 用户上传全开系统导出的 29 列 Excel 文件
- **THEN** 系统自动提取6个字段，校验数据，存储至 DailyReport 表，返回导入结果（成功条数、失败条数、失败原因）

#### Scenario: 其他投入清单模板导入
- **WHEN** 用户上传6列格式的 Excel 文件
- **THEN** 系统按统一逻辑解析，校验并存储，返回导入结果

#### Scenario: 导入数据校验
- **WHEN** 导入数据中存在日期格式错误、工时超范围（0-24h）、提交人不匹配
- **THEN** 系统标记该条数据为失败，在导入结果中列出失败原因

#### Scenario: 重复导入
- **WHEN** 用户重复导入同一日期数据
- **THEN** 系统提示是否覆盖

---

### Requirement: 工作量周报模块
系统 SHALL 基于日报数据自动汇总生成个人周报和团队周报，统计周期为上周四至本周四。

#### Scenario: 个人周报生成
- **WHEN** 用户导入日报数据后查看本周周报
- **THEN** 系统自动汇总上周四至本周四的日报数据，展示每日工时明细和本周总工时

#### Scenario: 提交周报
- **WHEN** 用户确认周报数据后点击提交
- **THEN** 系统记录提交时间，状态变为已提交，每周只能提交一次

#### Scenario: 撤回周报
- **WHEN** 用户撤回已提交的周报
- **THEN** 状态变为草稿，可重新提交

#### Scenario: 团队周报查看
- **WHEN** Leader 查看团队周报
- **THEN** 展示团队总工时、按项目维度统计、工作量排名、高负荷预警（>50h标红）、空缺提示（<20h标黄）

#### Scenario: 周报导出
- **WHEN** Leader 导出团队周报
- **THEN** 生成 Excel 文件，按周报参考模板格式（含组员上级、组员、所属框架、级别、软开工时、项目工时、总工时、结算单价、产能等列）

---

### Requirement: 月度工作量汇总模块
系统 SHALL 提供个人和团队的月度工作量汇总，支持图表展示和 Excel 导出。

#### Scenario: 个人月度汇总
- **WHEN** 用户查看月度汇总
- **THEN** 展示总投入时长、应工作天数、请假天数、日均投入、任务类型分布，配合图表展示

#### Scenario: 团队月度汇总
- **WHEN** Leader 查看团队月度汇总
- **THEN** 展示团队总工作量、按工作量排序的人员排名，支持按月/按人/按项目筛选

#### Scenario: 月度报表导出
- **WHEN** Leader 导出月度报表
- **THEN** 生成 Excel 文件

---

### Requirement: 请假管理模块
系统 SHALL 提供请假申请、直属上级审批、请假余额查询功能。

#### Scenario: 提交请假申请
- **WHEN** 员工填写请假类型、开始/结束日期、原因并提交
- **THEN** 系统按法定工作日计算请假天数，自动流转至直属上级审批

#### Scenario: 审批通过
- **WHEN** 直属上级点击通过
- **THEN** 请假状态变为已通过，通知申请人，请假记录关联工时统计

#### Scenario: 审批驳回
- **WHEN** 直属上级填写驳回原因并驳回
- **THEN** 请假状态变为已驳回，申请人可重新提交

#### Scenario: 请假余额查询
- **WHEN** 用户查询请假余额
- **THEN** 展示年假余额、本月已请假天数

---

### Requirement: 个人工作量看板
系统 SHALL 提供个人工作量看板，包含本周工时进度、月度累计、工作饱和度、项目工时占比等指标。

#### Scenario: 看板数据展示
- **WHEN** 用户访问个人看板
- **THEN** 展示本周已填报工时（进度条）、本月累计工时、请假天数、工作饱和度（高/中/低）、最近3条周报、各项目工时占比图表

#### Scenario: 月度日历
- **WHEN** 用户查看月度日历
- **THEN** 日历中标记正常工作日（绿色）、工作量不足（黄色）、请假（蓝色）、未填报（红色）、非工作日（灰色）、今日（高亮边框）

#### Scenario: 日历交互
- **WHEN** 用户点击日历某天
- **THEN** 展开当日工时明细

---

### Requirement: 团队人力空缺分析
系统 SHALL 结合请假、未填报、低工作量自动计算人力缺口，输出团队工作量空缺情况与人力建议。

#### Scenario: 人力缺口计算
- **WHEN** Leader 查看人力空缺分析
- **THEN** 系统汇总请假+未填报+低工作量人员，按条线/项目维度识别人力空缺，给出调配建议

---

### Requirement: UI 设计风格
前端 SHALL 采用 Dark Mode 科技感设计风格，参考 trae.cn 官网。

#### Scenario: 主题色彩
- **WHEN** 用户访问系统
- **THEN** 背景为近黑色（#0a0b0d），品牌色为亮绿（#32f08c），字体使用 Inter + JetBrains Mono

---

### Requirement: Docker 部署
系统 SHALL 支持前后端一体 Docker 镜像部署。

#### Scenario: Docker 部署
- **WHEN** 执行 docker-compose up
- **THEN** 前后端服务正常启动，数据库初始化完成，系统可访问
