# 团队工作量管理系统 · 需求文档

> 版本：v2.1 | 日期：2026-04-21 | 状态：正式发布 ## 

---

## 一、项目概述

**项目名称：** TeamWorkload（团队工作量管理系统）

**一句话描述：** 统一管理开发团队人力与工作量，自动统计工作量分布，识别人力饱和与空缺，支撑团队资源合理调配。

**目标用户：** 开发团队内部成员

| 角色 | 使用场景 |
| :--- | :--- |
| 普通成员 | 填报周报、查看个人看板、发起请假 |
| Leader / 项目经理 | 审批请假、查看团队汇总、识别人力饱和与空缺 |
| 管理员 | 用户管理、权限配置、系统维护 |

## 二、功能模块

### 2.1 工作量周报（个人 / 团队）

#### 2.1.1 周报统计周期

**统计范围：上周四至本周四**（每周四为团队周会时间点，周报统计周期与周会对齐）

| 配置项 | 说明 |
| :--- | :--- |
| 统计起始 | 上周四 00:00 |
| 统计截止 | 本周四 23:59 |
| 提交截止 | 每周四 18:00 前完成提交 |
| 周期标识 | 以截止日期所在周为周报标识（如：2026-W16） |

#### 2.1.2 日报数据来源

**统一通过模板导入**，不支持手动填写日报。所有日报数据均通过 Excel 清单导入方式录入系统。

**导入模板一：全开系统导入模板**

原始模板包含 29 列完整字段，系统导入时**自动提取以下 6 个字段**，其余字段忽略：

| 提取字段 | 说明 | 是否必填 |
| :--- | :--- | :--- |
| 所属项目/产品 | 项目分类（日常需求管理/日常运维管理） | 是 |
| 任务号 | 任务编号 | 是 |
| 任务名称 | 任务标题 | 是 |
| 工作日期 | 工作发生日期 | 是 |
| 实际工时 | 本次投入工时（小时） | 是 |
| 提交人 | 填报人姓名 | 是 |

> 原始模板其余字段（模块、系统、需求号、需求事项状态、需求名称、任务事项类型、工作描述、审批状态等）仅作展示参考，不参与系统数据存储。

**导入模板二：其他投入清单模板**

与全开系统导入模板采用统一字段格式，需按以下 6 个字段填写：

| 字段 | 说明 | 是否必填 |
| :--- | :--- | :--- |
| 所属项目/产品 | 项目分类（日常需求管理/日常运维管理） | 是 |
| 任务号 | 任务编号 | 是 |
| 任务名称 | 任务标题 | 是 |
| 工作日期 | 工作发生日期（YYYY-MM-DD） | 是 |
| 实际工时 | 本次投入工时（小时） | 是 |
| 提交人 | 填报人姓名 | 是 |

**导入规则：**
- 支持 .xlsx / .xls 格式
- 两种模板采用统一字段格式（所属项目/产品、任务号、任务名称、工作日期、实际工时、提交人），系统按统一逻辑解析
- 全开系统模板含 29 列，系统自动提取上述 6 个字段，其余忽略
- 导入前自动校验：日期格式、工时范围（0-24h）、提交人姓名与系统用户匹配
- 重复导入同一日期数据时，提示是否覆盖
- 导入结果反馈：成功条数、失败条数、失败原因明细

#### 2.1.3 个人周报

**功能描述：** 基于日报数据自动汇总生成个人周报，不支持手动补充调整。

| 字段 | 说明 |
| :--- | :--- |
| 姓名/工号 | 自动填充当前用户 |
| 填报周期 | 上周四 ~ 本周四 |
| 每日工时明细 | 按日展示各天工时（逐日拆分） |
| 本周总工时 | 自动汇总 |
| 提交状态 | 草稿 / 已提交 |
| 提交时间 | 自动记录 |

**操作流程：** 导入日报数据 → 确认汇总 → 保存草稿 → 提交  
**限制：** 每周只能提交一次，提交后不可修改（有权限可撤回）

#### 2.1.4 团队周报

**功能描述：** 项目经理可查看团队成员周报汇总，了解团队整体工作分布。

| 功能点 | 说明 |
| :--- | :--- |
| 团队本周总工时 | 所有成员本周工时汇总 |
| 按项目/模块维度统计 | 工时分布占比 |
| 工作量排名 | 本周投入时长从高到低排序 |
| 高负荷预警 | 单周 > 50h 标红预警 |
| 工作量空缺 | 本周 < 20h 标黄提示 |
| 时间筛选 | 本周/上周/自定义范围 |
| 导出 | 支持导出 Excel，格式参考周报模板（见下方） |

**周报导出模板格式：**

导出 Excel 按团队周会周报格式生成，参考 `周报参考模板_v1.0.xlsx`，包含以下列：

| 列名 | 说明 |
| :--- | :--- |
| 组员上级 | 成员归属上级（如：陈铭辛） |
| 组员 | 成员姓名 |
| 所属框架 | 项目/框架分类（核心常规/数柜项目/运维组等） |
| 级别 | 人员级别（初级/中级/高级） |
| 软开工时 | 本周软开工时（小时） |
| 项目工时 | 本周项目工时（小时） |
| 总工时 | 软开工时 + 项目工时 |
| 结算单价 | 按级别对应的结算单价 |
| 产能 | 总工时 × 结算单价 |
| 本周事项主要* | 本周主要工作事项 |
| 下周预计* | 下周预计工作安排 |
| 补充说明 | 请假、迟到等补充说明 |

### 2.2 月度工作量汇总（个人 / 团队）

#### 2.2.1 个人月度汇总

| 字段 | 说明 |
| :--- | :--- |
| 汇总周期 | 年月（YYYY-MM） |

#### 2.2.2 团队月度汇总

| :--- | :--- |

| 按工作量排序 | 展示最高/最低人员排名 |
| 多维度筛选 | 按月、按人、按项目 |
| 月度报表导出 | Excel 导出 |

### 2.3 请假管理

#### 2.3.1 请假登记（员工）

| 字段 | 说明 |
| :--- | :--- |
| 申请人 | 自动填充 |
| 请假类型 | 年假 / 病假 / 事假 / 婚假 / 产假 / 其他 |
| 开始日期 |  |
| 结束日期 |  |
| 请假天数 | 自动计算（按国家法定工作日计算，排除法定节假日与周末） |
| 请假原因 | 文本描述 |

**说明：** 无需上传附件，无需选择审批人（自动流转至直属上级）

#### 2.3.2 请假审批（直属上级）

**审批机制：** 直属上级审批，不支持多级审批。

| 操作 | 说明 |
| :--- | :--- |
| 通过 | 记录审批时间，更新请假状态 |
| 驳回 | 填写驳回原因 |
| 驳回后可重新提交 | 重新发起审批流程 |

**状态流转：** 草稿 → 审批中 → 通过 / 驳回  
**提醒：** 审批通过后自动通知申请人  
**关联：** 请假记录自动关联工时统计，扣除请假时段工时

#### 2.3.3 请假余额

| 功能点 | 说明 |
| :--- | :--- |
| 年假余额查询 | 剩余可用天数 |
| 本月已请假天数 | 看板展示 |
| 请假记录导出 | 支持统计报表 |

### 2.4 个人工作量看板

**核心目标：** 让员工一眼看清自己的投入情况

#### 2.4.1 看板指标

| 指标 | 说明 |
| :--- | :--- |
| 本周已填报工时 | 与本周总可用工时对比，进度条展示 |
| 本月累计工时 | 按周拆分展示 |
| 请假天数 | 本月已请假 / 剩余年假 |
| 工作饱和度 | 高（>50h）/ 中（30-50h）/ 低（<30h），可配置阈值 |
| 最近提交记录 | 最近 3 条周报列表 |
| 各项目工时占比 | 图表展示（类型自适应） |

#### 2.4.2 月度日历

**功能描述：** 在个人看板中嵌入月度日历视图，直观查看当月工作、请假、工作量不足情况。

| 日历标记 | 说明 | 颜色标识 |
| :--- | :--- | :--- |
| 正常工作日 | 已填报工时 ≥ 8h | 绿色 |
| 工作量不足 | 已填报工时 < 8h（工作日） | 黄色 |
| 请假 | 当日有请假记录 | 蓝色 |
| 未填报 | 工作日未录入任何工时 | 红色 |
| 非工作日 | 周末/法定节假日 | 灰色 |
| 今日 | 当前日期 | 高亮边框 |

**交互：** 点击日历某天，展开当日工时明细与工作描述

---

### 2.5 团队人力空缺分析

**功能描述：** 结合请假、未填报、低工作量自动计算人力缺口，输出团队工作量空缺情况与人力建议。

| 功能点 | 说明 |
| :--- | :--- |
| 人力缺口计算 | 请假 + 未填报 + 低工作量人员汇总 |
| 空缺预警 | 按条线/项目维度识别人力空缺 |
| 人力调配建议 | 基于工时数据给出资源调配参考 |

---

### 2.6 工作日管理

**功能描述：** 系统对接互联网工作日接口，自动获取国家法定工作日，用于工时计算与请假天数计算。

| 配置项 | 说明 |
| :--- | :--- |
| 数据来源 | 对接互联网工作日接口（如百度/聚合数据等免费API） |
| 同步频率 | 每年自动同步下一年度工作日数据 |
| 手动调整 | 管理员可手动标记调休工作日/补班日 |
| 影响范围 | 请假天数计算、日历标记、工时饱和度计算 |

## 三、非功能需求

### 3.1 性能需求

| 指标 | 要求 |
| :--- | :--- |
| 页面加载 | ≤ 2秒 |
| 数据查询 | ≤ 3秒 |
| 并发用户数 | ≥ 50人同时在线 |
| 数据导出 | ≤ 10000条，导出时间 ≤ 10秒 |
| Excel导入 | 单次导入 ≤ 5000条，导入时间 ≤ 15秒 |

### 3.2 安全需求

| 需求 | 描述 |

| 权限控制 | 基于角色的权限管理，数据隔离到个人/团队级别 |

## 五、数据模型

### 5.1 核心实体

| 中文名 | 英文名 | 说明 |
| :--- | :--- | :--- |
| 用户 | User | 系统用户，独立管理（含级别、结算单价、归属上级） |
| 角色 | Role | 用户角色（普通成员/Leader/管理员） |
| 项目 | Project | 项目信息 |
| 日报 | DailyReport | 每日工作记录（逐日拆分） |
| 周报 | WeeklyReport | 个人周报主表 |
| 月度汇总 | MonthlySummary | 个人月度汇总（计算生成） |
| 请假申请 | LeaveRequest | 请假单主表 |
| 工作日 | WorkDay | 法定工作日/节假日配置 |

### 5.2 核心表结构

**User（用户）**
```
id, username, password, name, dept, role, level(初级/中级/高级),
unit_price(结算单价), leader_id(归属上级), email, status, create_time
```

> 归属上级示例：负责人-陈铭辛，组员-杨江萍（杨江萍的 leader_id 指向陈铭辛）

**DailyReport（日报）**
```
id, user_id, work_date, project_name, task_no, task_name,
hours, source(quankai/other), import_batch_no, create_time, update_time
```

**WeeklyReport（周报）**
```
id, user_id, week_start_date(周四), week_end_date(周四), total_hours,
status(submitted/draft), submitted_at, create_time, update_time
```

**MonthlySummary（月度汇总）**
```
id, user_id, year_month, total_hours, work_days(应工作天数), leave_days,
daily_avg, task_type_dist(JSON), create_time
```

**LeaveRequest（请假）**
```
id, user_id, leave_type, start_date, end_date,
days, reason, approver_id(直属上级), status(pending/approved/rejected),
reject_reason, create_time, approve_time
```

**WorkDay（工作日）**
```
id, date, is_workday, type(workday/weekend/holiday/makeup), description, year,
create_time
```
