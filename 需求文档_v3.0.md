# 团队工作量管理系统 · 需求文档

> 版本：v3.0 | 日期：2026-04-24 | 状态：正式发布（与代码完全对齐）

---

## 一、项目概述

**项目名称：** TeamWorkload（团队工作量管理系统）

**一句话描述：** 统一管理开发团队人力与工作量，自动统计工作量分布，识别人力饱和与空缺，支撑团队资源合理调配。

**目标用户：** 开发团队内部成员

| 角色 | 使用场景 |
| :--- | :--- |
| 普通成员（MEMBER） | 填报日报、查看个人看板、发起请假、查看周报 |
| 负责人（LEADER） | 审批请假、查看团队汇总、月度汇总、识别人力饱和与空缺 |
| 管理员（ADMIN） | 用户管理、权限配置、日报导入、工作日管理、系统维护 |

> **特殊规则：** 管理员用户不参与业务统计，所有请假审批统计、团队周报详情、月度汇总等模块自动排除 ADMIN 角色用户的数据。

## 二、功能模块

### 2.1 工作量周报（个人 / 团队）

#### 2.1.1 周报统计周期

**统计范围：上周四至本周三**（每周四为团队周会时间点，周报统计周期与周会对齐）

| 配置项 | 说明 |
| :--- | :--- |
| 统计起始 | 上周四 00:00 |
| 统计截止 | 本周三 23:59 |
| 提交截止 | 每周四 18:00 前完成提交 |
| 周期标识 | 以截止日期（本周三）作为 `weekEndDate` 唯一标识 |

> **关键规则：** 本周四当天数据不计入本周统计，自动归入下一周。例如：2026-04-23（周四）的日报数据归入 2026-04-23~2026-04-29 这一统计周期。

#### 2.1.2 日报数据来源

系统支持两种日报录入方式：

**方式一：Excel 模板导入（管理员操作）**

支持两种 Excel 模板格式：

| 模板类型 | 说明 | 提取字段 |
| :--- | :--- | :--- |
| 全开系统导入模板 | 原始模板含 29 列，系统自动提取 6 个字段 | 所属项目/产品、任务号、任务名称、工作日期、实际工时、提交人 |
| 其他投入清单模板 | 统一 6 列格式 | 所属项目/产品、任务号、任务名称、工作日期、实际工时、提交人 |

**导入规则：**
- 支持 .xlsx / .xls 格式
- 两种模板采用统一字段格式，系统按统一逻辑解析
- 全开系统模板含 29 列，系统自动提取上述 6 个字段，其余忽略
- 导入前自动校验：日期格式（支持 8 种格式）、工时范围（0-24h）、提交人姓名与系统用户匹配、用户状态检查
- 重复数据检测：同一 userId + workDate + taskNo 视为重复，放入待确认列表
- 确认覆盖：管理员确认后，删除旧记录并插入新记录
- 导入结果反馈：成功条数、失败条数、失败原因明细

**方式二：手动录入（所有用户操作）**

| 字段 | 说明 | 是否必填 |
| :--- | :--- | :--- |
| 所属项目/产品 | 下拉选择（可搜索可创建），默认选项为"日常需求管理""日常运维管理" | 是 |
| 任务号 | 手动输入 | 是 |
| 任务名称 | 手动输入 | 是 |
| 工作日期 | 日期选择器，格式 YYYY-MM-DD | 是 |
| 实际工时 | 数字输入，步长 0.5，范围 0.5-24 | 是 |

> 手动录入的日报 `source` 标记为 `OTHER`，导入的日报标记为 `QUANKAI` 或其他来源。手动录入不做重复检测。

#### 2.1.3 团队周报详情

**功能描述：** 展示当前登录人及其下属的周报数据，支持按周翻页查看。

**表格列定义：**

| 列名 | 说明 | 对齐方式 |
| :--- | :--- | :--- |
| 负责人 | 成员归属上级（若上级为 ADMIN 则显示自己） | 左对齐 |
| 组员 | 成员姓名 | 左对齐 |
| 本周事项 | 去重后的任务列表，每条带序号分行展示（1. 任务A 2. 任务B） | 左对齐 |
| 总工时（h） | 纯数字，不带单位 | 居中 |
| 下周预计事项 | 可编辑输入框 / 只读文本，列标题动态显示对应周期 | 左对齐 |
| 补充说明 | 可编辑输入框 / 只读文本 | 左对齐 |
| 操作 | 提交按钮 / 已提交标签 / 仅查看标签 | 居中 |

**本周事项展示规则：**
- 后端返回换行符分隔的任务文本
- 前端按换行符、逗号（中英文）、分号（中英文）分割
- 去重：完全相同的任务文本仅保留 1 条
- 自动添加 1. 2. 3. 格式序号
- 每个任务单独占一行，行高自适应

**按周分页功能：**
- 分页控件位于表格上方，显示当前周范围（格式：YYYY-MM-DD ~ YYYY-MM-DD）
- 支持上一周 / 下一周按钮切换
- 切换后表格数据同步更新为对应周的内容
- 默认展示本周数据

**下周预计事项列标题动态显示：**
- 列标题格式：`下周预计事项（本周四～下周三）`
- 示例：`下周预计事项（2026-04-23～2026-04-29）`

**权限与按钮状态规则：**

| 周模式 | 判定条件 | 下周预计事项 | 补充说明 | 操作列 |
| :--- | :--- | :--- | :--- | :--- |
| 本周（current） | 查看周范围 = 当前真实周范围 | 可编辑 | 可编辑 | 显示「提交」按钮 |
| 下周（next） | 查看周起始日 > 当前真实周起始日 | 可编辑 | 只读 | 显示「填写中」标签 |
| 历史周（history） | 查看周起始日 < 当前真实周起始日 | 只读 | 只读 | 显示「仅查看」标签 |

> **额外约束：** 只有当前行的负责人（leader 字段）等于当前登录用户名时才可编辑；已提交（SUBMITTED）的行所有字段只读。

**提交按钮状态：**
- 未提交：显示「提交」按钮（可点击）
- 已提交：显示「已提交」文字标签（绿色，不可点击）
- 提交前自动保存输入框内容，再调用提交接口

**导出详情功能：**
- 导出数据与页面当前周范围强绑定
- 文件名格式：`团队周报详情_YYYY-MM-DD~YYYY-MM-DD.xlsx`
- 切换周范围后再次导出，数据和文件名同步更新

#### 2.1.4 个人周报

**功能描述：** 基于日报数据自动汇总生成个人周报。

| 字段 | 说明 |
| :--- | :--- |
| 姓名 | 自动填充当前用户 |
| 填报周期 | 上周四 ~ 本周三 |

| 提交状态 | DRAFT / SUBMITTED |

**操作流程：** 导入/录入日报数据 → 填写下周预计事项和补充说明 → 提交周报

**限制：** 每周只能提交一次，提交后不可修改（有权限可撤回）

### 2.3 请假管理

#### 2.3.1 请假登记（员工）

| 字段 | 说明 | 是否必填 |
| :--- | :--- | :--- |
| 申请人 | 自动填充当前用户 | 是 |
| 请假类型 | 年假(ANNUAL) / 病假(SICK) / 事假(PERSONAL) / 婚假(MARRIAGE) / 产假(MATERNITY) / 其他(OTHER) | 是 |
| 开始日期 | 日期选择器 | 是 |
| 结束日期 | 日期选择器 | 是 |
| 开始时段 | 上午 / 下午 | 是 |
| 结束时段 | 上午 / 下午 | 是 |
| 请假天数 | 自动计算（精确到 0.5 天，按国家法定工作日计算，排除法定节假日与周末） | 自动 |
| 请假原因 | 文本描述 | 是 |

**日期禁用规则：**
- 优先查询 WorkDay 表数据：`isWorkday !== 1` 的日期禁用
- 无数据时默认禁用周末（周六、周日）

**时间重叠检查：**
- 前端和后端均做重叠检查
- 将每天划分为上午(1)、下午(2)、全天(3)三个时段编码
- 检查是否存在时段冲突，精确到半天

**请假天数计算规则（精确到半天）：**
- 查询 WorkDay 表判断每天是否为工作日
- 上午判定：开始时间小时 < 12
- 下午判定：结束时间小时 >= 13
- 同一天：上午到下午算 1 天，仅上午或仅下午算 0.5 天
- 开始日期：上午开始算 1 天，下午开始算 0.5 天
- 结束日期：下午结束算 1 天，上午结束算 0.5 天
- 中间完整工作日算 1 天
- 最终结果四舍五入保留 1 位小数

**年假余额规则：**
- 提交年假时立即扣除年假余额
- 可用年假 = 当前余额 - 待审批年假天数
- 余额不足时不允许提交
- 驳回或撤回时归还年假余额

**余额展示：**
- 年假余额：当前年假余额
- 本月已请天数：本月已审批通过的请假天数总和

#### 2.3.2 请假审批（直属上级）

**审批机制：** 直属上级审批，不支持多级审批。用户必须设置了直属上级（leaderId）才能提交请假。

| 操作 | 说明 |
| :--- | :--- |
| 通过 | 记录审批时间，更新请假状态为 APPROVED |
| 驳回 | 填写驳回原因，归还年假余额（如为年假），状态改为 REJECTED |
| 撤回 | 仅申请人可操作，仅 PENDING 状态可撤回，归还年假余额（如为年假），状态改为 REVOKED |

**状态流转：** PENDING → APPROVED / REJECTED / REVOKED

**审批页面功能：**
- 待审批列表：展示待审批请假，含通过/驳回按钮
- 月度请假趋势图：ECharts 柱状图，按人汇总已通过请假天数，>3 天红色渐变，≤3 天绿色渐变
- 历史审批记录：支持按申请人、请假类型、状态筛选，前端分页

**统计排除规则：** 所有请假审批统计（待审批列表、历史记录、趋势图）自动排除 ADMIN 角色用户的请假数据。

#### 2.3.3 请假余额

| 功能点 | 说明 |
| :--- | :--- |
| 年假余额查询 | 当前余额、待审批年假天数、可用年假 |
| 本月已请假天数 | 看板展示 |

### 2.6 团队人力空缺分析

**功能描述：** 结合请假、未填报、低工作量自动计算人力缺口。

| 功能点 | 说明 |
| :--- | :--- |
| 请假中 | 本周有已批准请假的成员 |
| 未填报 | 本周工时为 0 的成员 |
| 低工时 | 本周工时 > 0 但 < 20h 的成员 |
| 空缺率计算 | 空缺人数 / 团队总人数 |
| 人力调配建议 | 空缺率 >30% 建议协调资源；>10% 建议关注状态；≤10% 正常范围；0% 饱满 |

**统计排除规则：** 自动排除 ADMIN 角色用户。

## 五、数据模型

### 5.1 核心实体

| 中文名 | 英文名 | 说明 |
| :--- | :--- | :--- |
| 用户 | SysUser | 系统用户，独立管理（含级别、结算单价、归属上级） |
| 日报 | DailyReport | 每日工作记录（逐日拆分） |
| 周报 | WeeklyReport | 个人周报主表 |
| 月度汇总 | MonthlySummary | 个人月度汇总（计算生成） |
| 请假申请 | LeaveRequest | 请假单主表 |
| 工作日 | WorkDay | 法定工作日/节假日配置 |
| 项目 | Project | 项目信息 |
| 权限 | Permission | 细粒度权限控制 |

### 5.2 核心表结构

**SysUser（用户）**
```
id, username, password(明文), name, dept, role(MEMBER/LEADER/ADMIN),
level(初级/中级/高级), unit_price(结算单价), leader_id(归属上级),
email, annual_leave_balance(年假余额), status(0禁用/1启用), create_time
```

**DailyReport（日报）**
```
id, user_id, work_date, project_name, task_no, task_name,
hours, source(QUANKAI/OTHER), import_batch_no, create_time, update_time
```

**WeeklyReport（周报）**
```
id, user_id, week_start_date(上周四), week_end_date(本周三), total_hours,
next_week_plans, notes, status(DRAFT/SUBMITTED), submitted_at, create_time, update_time
```

**MonthlySummary（月度汇总）**
```
id, user_id, year_month, total_hours, work_days(应工作天数), leave_days,
daily_avg, project_dist(JSON), create_time
```

**LeaveRequest（请假）**
```
id, user_id, leave_type(ANNUAL/SICK/PERSONAL/MARRIAGE/MATERNITY/OTHER),
start_date, end_date, start_period, end_period,
days(精确到0.5天), reason, approver_id(直属上级),
status(PENDING/APPROVED/REJECTED/REVOKED), reject_reason, create_time, approve_time
```

**WorkDay（工作日）**
```
id, date, is_workday(0/1), type(workday/weekend/holiday/makeup), description, year,
create_time
```

## 六、UI 设计风格

设计风格参考 **trae.cn** 官网，科技感 Dark Mode：

| 元素 | 规范 |
| :--- | :--- |
| 背景 | `#0a0b0d`（近黑）/ `#121314`（次级深灰） |
| 品牌色 | `#32f08c`（亮绿），渐变 `#3ee1a3 → #32f08c → #60f2bd → #a0fde7` |
| 字体 | Inter（正文）+ JetBrains Mono（数据/代码） |
| 圆角 | 6px（小）/ 12px（卡片）/ 16px（大） |
| 边框 | `rgba(255,255,255,0.08)` 多层级半透明白边 |
| 最大宽度 | 1600px，响应式断点 768 / 1280 / 1920px |
| 动效 | hover 过渡 150ms，`scale(1.02)` |
| 日历颜色 | 绿色(正常) / 黄色(不足) / 蓝色(请假) / 红色(未填报) / 灰色(非工作日) |

**表格样式规范（深色主题统一）：**
- 表头背景色：`$bg-secondary`，文字色：`$text-secondary`

- 边框颜色：`$border-color`（`rgba(255, 255, 255, 0.06)`）
- 输入框背景色：`$bg-input`，边框色：`$border-color`
- 提交按钮：品牌主色 `$brand-primary`（#32f08c），禁用态灰色
